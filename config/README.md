# 内容配置说明

这个文件夹用于维护网站的结构化内容。当前网站仍然是静态 HTML；这些 YAML 文件先作为“内容源”，你改完后可以让我扫描配置并同步到页面。

推荐维护顺序：

1. 先改 `projects.yaml`，补项目条目、负责人、参与者、生命周期和成果。
2. 再改 `members.yaml`，维护顾问组、正式队员、梯队队员和往届名单。
3. 再改 `talent.yaml`，维护人才去向页面的教育/就业履历。
4. 资源类详情统一写在 `resources.yaml`，后续用于生成培养计划、设备清单等子页面。

字段可以先留空，但建议不要删除字段名。这样后续同步页面时不容易漏信息。
